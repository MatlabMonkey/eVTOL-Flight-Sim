eVTOL Project Handoff

Date: March 24, 2026

This file is meant to get the group up to speed quickly on the current state of the multi-rotor eVTOL flight-simulation project and on what changed for Progress Report 2. The report itself should live in docs/Brown Progress Report 2.pdf once saved.

What Was Completed in This Update

  • The aircraft setup was cleaned up so the aircraft definition and scenario setup are now separated into aircraft_def.m, scenario_def.m, and Full_Sim_Init.m.
  • The vehicle geometry was updated to a more defensible Archer Midnight-like lift+cruise layout with two wing halves, a V-tail, six front tilting rotors, and six rear fixed rotors.
  • The mass properties and aerodynamic coefficients were updated using NASA lift+cruise reference material instead of earlier placeholder values.
  • Control surfaces were added in the aircraft definition and visualization: one flaperon per wing half and one ruddervator per V-tail half.
  • The rendering tool was improved so it shows rotor thrust vectors, aerodynamic surface normals, and control-surface normals. This was used to verify geometry and sign conventions during development.
  • A simple PD-controller scaffold was added to the Simulink model so there is now a closed-loop path from vehicle states to aerodynamic control commands and rotor commands, but it is still not tuned.
  • Transition metrics were narrowed to transition time and RMS jerk so later controller comparisons can be made on a consistent basis.

Key Files to Know

  • Full_Sim_Init.m: main setup script used to initialize the simulation workspace.
  • aircraft_def.m: aircraft geometry, masses, inertia, aerodynamic coefficients, propulsion layout, and control-surface metadata.
  • scenario_def.m: initial conditions, commands, and scenario-level setup.
  • render_aircraft.m: 3D render used to verify aircraft geometry, thrust directions, rotor tilt, and control-surface deflections.
  • Brown_Full_Sim.slx: main simulation model.
  • TESTER_external_forces_updated.mlx: updated validation/test live script.
  • docs/AME532_Progress_Report_Draft_1_BrownStyle_2026-03-24_revised.html: shortened report draft used as the basis for Progress Report 2.

What Has Been Verified

  • Full_Sim_Init.m runs successfully.
  • Brown_Full_Sim.slx updates and simulates successfully in its current baseline configuration.
  • The updated tester workflow runs using the current aircraft values.
  • The trim helper runs and returns usable first-pass actuator estimates.
  • The example transition-metrics script runs and outputs the selected metrics.
  • The render script correctly shows hover, transition, and cruise rotor tilt, plus flap and ruddervator deflections.

What Is Not Done Yet

  • The PD controller is only a scaffold right now. It executes, but it is not tuned and should not be presented as a finished controller.
  • The current trim result is a first-pass estimate, not a fully converged 6-DoF trim solution.
  • The next major technical step is closed-loop controller tuning and then running controlled transition cases with the selected metrics.

Commands the Group Can Run

Initialize the project:

run('Full_Sim_Init.m')

Render a transition case with all aerodynamic surfaces visibly deflected down:

render_aircraft('transition', 45, 'delta_f_deg', 15, 'delta_e_deg', 15)

Run the updated validation live script:

run('TESTER_external_forces_updated.mlx')

Run the example transition metrics:

run('scripts/metrics/example_metrics_run.m')

Run the trim check:

run('scripts/trim/run_trim_check.m')

What to Say in the Report

The cleanest summary is that the project moved from a placeholder VTOL model toward a more defensible aircraft-specific simulation. The aircraft properties were updated using NASA lift+cruise references, control surfaces were added to the geometry and model structure, a simple PD-controller scaffold was integrated, and transition metrics were selected so later controller work can be evaluated consistently. The most important thing to keep honest is that controller tuning and successful controlled transition are still future work.
